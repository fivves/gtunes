# gTunes 1.1.4

gTunes 1.1.4 is The Stability Update for the Linux Jellyfin music client.

This is not a big feature release. It focuses on making the core of the app
more robust.

## Highlights

- Move more playback, queue, radio, and restore logic into the playback session
  so state transitions stay centralized and predictable.
- Add targeted playback session test coverage around restore ordering and
  helper behavior.
- Preserve the current playback position during transcode fallback so stream
  recovery is less disruptive.

## Fixes and Refinements

- Decouple Discord artwork uploads and presence updates from UI cache and MPRIS
  wiring to reduce cross-system playback side effects.
- Simplify the app window title and refine keyboard shortcut support, including
  a direct shuffle toggle.
- Update release-facing documentation and screenshots to match the current app.

## Release Artifact

The AppImage attached to the GitHub release is built from the release profile
and named with the package version from `Cargo.toml`. It packages the gTunes
binary and desktop metadata so users do not need the Rust toolchain or native
development packages to launch it.

The AppImage still expects a Linux desktop with GTK4, Libadwaita, GStreamer,
and the required GStreamer runtime plugins available.
