# gTunes 1.1.3

gTunes 1.1.3 is The Next-Up Update for the Linux Jellyfin music client.

## Highlights

- Persist the active playback queue so gTunes can restore the current track,
  queue order, position, and shuffle state after restart.
- Add Discord Rich Presence for the currently playing track, including title,
  artist, album, album art, and playback timer.
- Remove the position-based playback stall watchdog that could incorrectly
  restart a track from the beginning during background activity.

## Fixes and Refinements

- Document Discord Rich Presence setup, optional app art, and artwork privacy
  behavior.
- Keep the roadmap aligned with the current playback and architecture
  priorities.

## Release Artifact

The AppImage attached to the GitHub release is built from the release profile
and named with the package version from `Cargo.toml`. It packages the gTunes
binary and desktop metadata so users do not need the Rust toolchain or native
development packages to launch it.

The AppImage still expects a Linux desktop with GTK4, Libadwaita, GStreamer,
and the required GStreamer runtime plugins available.
